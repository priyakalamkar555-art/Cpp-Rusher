# OOP Concepts in C++ #

1. What is an Object Oriented Programming ?

Object-Oriented Programming (OOP) is a programming approach that organizes a program around objects 
and classes instead of functions and logic alone.
An object represents a real-world entity (like Student, Car, BankAccount) and contains:

● Data members (variables)
● Member functions (methods)

C++ supports both procedural and object-oriented programming.

2. Classes in C++

A class is a user-defined data type that acts as a blueprint for creating objects.
Characteristics:

● Contains data and functions
● Does not occupy memory until objects are created

 
● Example:
class Student {
public:
    int roll;
    string name;

    void show() {
        cout << roll << " " << name;
    }
};

3. Objects in C++

An object is an instance of a class.
Memory is allocated when an object is created.

 
        Student s1;
        s1.roll = 1;
        s1.name = "Aman";
        s1.show();

4. Encapsulation

Encapsulation means binding data and functions together and protecting data from outside access.

Achieved using:

● Classes
● Access specifiers

Benefits:

● Data security
● Controlled access
● Better maintainability

 
● Example:

class Bank {
private:
    int balance;
public:
    void setBalance(int b) {
        balance = b;
    }
    int getBalance() {
        return balance;
    }
};

5. Abstraction

Abstraction means hiding internal implementation details and showing only what is necessary.

● Example:

When you use cout, you don’t know how it prints internally — that’s abstraction.

Achieved using:

● Classes
● Header files
● Access specifiers

6. Inheritance

Inheritance allows one class to reuse properties and functions of another class.

 
● Syntax:

class Child : public Parent {
};

● Example:

class Animal {
public:
    void eat() {
        cout << "Eating";
    }
};

class Dog : public Animal {
public:
    void bark() {
        cout << "Barking";
    }
};


● Types of Inheritance:

1. Single Inheritance
2. Multiple Inheritance
3. Multilevel Inheritance
4. Hierarchical Inheritance
5. Hybrid Inheritance

7. Polymorphism

Polymorphism means “many forms”.
It allows the same function or operator to behave differently.

● Types:

1. Compile-time Polymorphism
2. Run-time Polymorphism

8. Function Overloading (Compile-time Polymorphism)

Function overloading means multiple functions with the same name but different parameters.

● Rules:

1. Different number of arguments OR
2. Different data types

 
● Example:

int add(int a, int b) {
    return a + b;
}

float add(float a, float b) {
    return a + b;
}


✔ Decision is made at compile time.

9. Operator Overloading

Operator overloading allows operators to work with user-defined data types.

 
● Example:

class Test {
public:
    int x;
    Test(int a) {
        x = a;
    }

    Test operator +(Test t) {
        return Test(x + t.x);
    }
};

10. Run-time Polymorphism

Occurs when a derived class overrides a base class function.

Uses:

● Virtual functions
● Base class pointer

 
class Base {
public:
    virtual void show() {
        cout << "Base class";
    }
};

class Derived : public Base {
public:
    void show() {
        cout << "Derived class";
    }
};


✔ Function call resolved at run time

11. Virtual Function

A function declared using virtual keyword to support runtime polymorphism.
    
    virtual void display();

12. Constructor

constructor is a special function with the same name as the class. It is called automatically when an object is created.

Types:

1. Default Constructor
2. Parameterized Constructor
3. Copy Constructor

 
class Demo {
public:
    Demo() {
        cout << "Constructor called";
    }
};

13. Destructor

A destructor is used to free resources. Called automatically when object goes out of scope.

 
~Demo() {
    cout << "Destructor called";
}

Advantages of OOP

✔ Code reuse
✔ Better security
✔ Easy maintenance
✔ Real-world modeling

Disadvantages of OOP

✖ More memory usage
✖ Slower execution
✖ Complex design
